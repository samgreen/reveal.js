What is Node?

Node is a server side implementation of Google's V8 JavaScript engine.

It's a blazing fast event driven framework. It's non-blocking and asynchronous.

Similar to Twisted (Python) and EventMachine (Ruby).

Handles I/O in a very special / unique way. I want to focus on these three terms throughout this presentation. Event Driven - This means that the program flow is altered from internal or external events. Other modern systems use polling and threads as an alternative to events. Non Blocking - This means that the program and the libraries it makes use of will not block the main event loop. If the main event loop is blocked, no events can be delivered until the current function returns control to the loop. Asynchronous - This means that program code is executed in parallel and in the background.

What about multi-core servers?

Aren't threads necessary to scale programs to multi-core computers?

You can start new processes via child_process.fork() these other processes will be scheduled in parallel.

For load balancing incoming connections across multiple processes use the cluster module.

Who is using Node?

Synchronous JavaScript


// This will BLOCK the event loop!
// result - contains the query result
var result = db.query("SELECT ...");

Asynchronous JavaScript


db.query("SELECT ...", function (err, result) {
   // err - null, or contains error details
   // result - contains the query result
});

We're taught to write blocking code


var result = db.query("SELECT ...");

The problem with this approach is scalability. While this code is easy to write and simple to comprehend, it's almost impossible to scale.
